# Contributing

## Before opening a change

- Keep changes focused and add or update tests for changed behavior.
- Run backend tests, frontend tests and the frontend build.
- Do not add generated files, local databases, assets, `.env` files, secrets or operational logs.
- Keep user-visible strings in the locale files.

## Security

Never include credentials, access tokens, private hosts, personal data or production paths in commits, test fixtures, screenshots, issues or pull requests. See [SECURITY.md](SECURITY.md) for vulnerability reporting.
